hexo博客中使用github作为图片的链接源

今天在用马克飞象写博客的时候，直接向博客中插入了截图，这个在本地运行很好，可以生成pdf和markdown文件。但是当将该博客提交到github上时，图片却没有显示。网上搜索了一些解决方法，pein0119的方法恰好解决了我的问题，现在总结一下，方便以后查阅。

  1. 首先在source目录下新建一个目录，如image，博客中需要插入的图片放到该目录下。执行hexo d时，hexo会将这些图片push到github上。

  2. 在博客中通过！[链接文字](/path/image)命令插入图片。假如我有一张图片在github上的地址为https://github.com/skycsliuyang/skycsliuyang.github.io/blob/master/img/1465786546512.png，在markdown文件中插入该图片就是用下面的命令：
    ![](https://github.com/skycsliuyang/skycsliuyang.github.io/blob/master/img/1465786546512.png?raw=true)

    注意：不要漏掉末尾的?raw=true。

Comnoid 静态污点分析工具

一个基于FlowDroid的静态污点分析工具，它可以提供更加精确的静态污点分析，并且可以分析应用程序间数据泄露。

组成

基于FlowDroid，主要包含两个工具：
ApkGrabber：收集用户手机上的应用程序。

Comnoid：接收apk输入，输出数据泄露报告。

Comnoid相比FlowDroid可以检测跨应用的信息泄露。主要增加了一个Inter app 处理模块。

Permission Mapper：
抽取应用程序使用的权限。并将其归类成四种。

Intent Analyser
主要检测implicit Intent，因为implicit Intent可能被恶意应用获取。当携带弱权限或者无权限限制implicit Intent存在，则存在app之间的数据泄露。

Component Analyzer Moudle
从manifest文件和Dalvik指令中抽取component的信息。这些信息用来检查是否设置EXPORTED flag或者是否提供了intent filters。
如果一个exported component注册了一个non-Android action string，并且也发现component发出携带相同action string的implicit intent，则Comnoid报告一个数据泄露。
exported broadcast receivers报告一个broadcast injection leak，exported activities报告一个activity launch attack，exported services报告一个service launch attack。

解决NTFS移动硬盘在Ubuntu无法挂载的问题

今天向移动硬盘复制大量文件的时候，Ubuntu突然没有了反应，因此强行把移动硬盘拔了下来，结果悲剧了，再插上移动硬盘出现了下面这个错误。

"Error mounting /dev/sdb1 at /media/fuzzy27/My Book: Command-line `mount -t "ntfs" -o 
"uhelper=udisks2,nodev,nosuid,uid=1000,gid=1000,dmask=0077,fmask=0177" "/dev/sdb1" 
"/media/fuzzy27/My Book"' exited with non-zero exit status 13: $MFTMirr does not match $MFT (record 0).
Failed to mount '/dev/sdb1': Input/output error
NTFS is either inconsistent, or there is a hardware fault, or it's a
SoftRAID/FakeRAID hardware. In the first case run chkdsk /f on Windows
then reboot into Windows twice. The usage of the /f parameter is very
important! If the device is a SoftRAID/FakeRAID then first activate
it and mount a different device under the /dev/mapper/ directory, (e.g.
/dev/mapper/nvidia_eahaabcc1). Please see the 'dmraid' documentation
for more details."

当出现这种问题时，解决方式正如提示给出的那样，在这里我把步骤详细说一下：
注：2到6步都是在Windows上进行。

  1. 从Linux上移除移动硬盘
  2. 将该移动硬盘插到运行Windows的电脑上
  3. 以管理员方式打开命令行窗口
  4. 进入该移动硬盘的根目录，如盘符为K，则进入K盘的根目录
  5. 输入chkdsk /f，回车，此时会有提示，显示目录正在被使用，重启时会检查该盘，balabala的还有一些，不用管他
  6. 不要拔出设备，保存其他工作后，直接重启。再次启动时会检查并修复磁盘的错误
  7. 修复完成后，该移动硬盘就可以正常挂载到Ubuntu中
